Python scientific computing ecosystem

1.1.1 The scientist‚Äôs needs
‚Ä¢ Get data (simulation, experiment control),
‚Ä¢ Manipulate and process data,
‚Ä¢ Visualize results, quickly to understand, but also with high quality figures, for reports or publica- tions.
1.1.2 Python‚Äôs strengths
‚Ä¢ Batteries included Rich collection of already existing bricks of classic numerical methods, plot- ting or data processing tools. We don‚Äôt want to re-program the plotting of a curve, a Fourier transform or a fitting algorithm. Don‚Äôt reinvent the wheel!
‚Ä¢ Easy to learn Most scientists are not payed as programmers, neither have they been trained so. They need to be able to draw a curve, smooth a signal, do a Fourier transform in a few minutes.
‚Ä¢ Easy communication To keep code alive within a lab or a company it should be as readable as a book by collaborators, students, or maybe customers. Python syntax is simple, avoiding strange symbols or lengthy routine specifications that would divert the reader from mathematical or scientific understanding of the code.
‚Ä¢ Efficient code Python numerical modules are computationally efficient. But needless to say that a very fast code becomes useless if too much time is spent writing it. Python aims for quick development times and quick execution times.
‚Ä¢ Universal Python is a language used for many different problems. Learning Python avoids learning a new software for each new problem.

2.4.4 Passing by value
----------------------
Tip: Can you modify the value of a variable inside a function? Most languages (C, Java, . . . ) distinguish ‚Äúpassing by value‚Äù and ‚Äúpassing by reference‚Äù. In Python, such a distinction is somewhat artificial, and it is a bit subtle whether your variables are going to be modified or not. Fortunately, there exist clear rules.
Parameters to functions are references to objects, which are passed by value. When you pass a variable to a function, python passes the reference to the object to which the variable refers (the value). Not the variable itself.
If the value passed in a function is immutable, the function does not modify the caller‚Äôs variable. If the value is mutable, the function may modify the caller‚Äôs variable in-place:
